Create a Status

Within the Workflow State Manager Configuration screen, click New to create a new Workflow Status, enter the required fields and click Save.

Field

Description

Status (Required)

Enter a descriptive label to summarize the purpose of the Status.

Is Start Node (Read Only)

The system will mark this checkbox only after Status Transitions have been defined and used to easily identify the configured starting Status.

Is End Node (Read Only)

The system will mark this checkbox only after Status Transitions have been defined and used to easily identify the configured ending Status before the system takes over.

Completion Days Out (Optional)

The number of days prior from the solution object’s goal (i.e. Price & Promotion = Start Date, Assortment = Receipt Date). The system will use this date to display a Status’ ‘due date’ but will not take any action on it unless a Trigger Method is defined.

Negative values (indicating days after the goal completion date) are allowed. For example, to create a status for Price & Promotion that is due 5 days after the start date, enter “-5” as the Completion Days Out.

Prefix Method (Optional)

The dropdown is populated with available methods (i.e. Advanced processes applied at the database level) used to execute configured system level actions prior to applying the Status.

Postfix Method (Optional)

The dropdown is populated by the available methods (i.e. Advanced processes applied at the database level) used to execute configured system level actions after applying the Status.

Trigger Method (Optional)

The dropdown is populated with actions to execute when the Completion Days Out is equal to or less than the completion date. The Trigger Method is used to advance to a ‘next’ Status and validations or additional actions may be configured within it.

Notes:

  • If multiple ‘next’ Statuses exist, the Trigger Method should contain a process to determine which ‘next’ Status is the most appropriate using the Transition ‘Weight’ column.
  • If a status has negative Completion Days Out values, the Workflow State Manager Heartbeat Task (see page 1) stops checking the status on the furthest negative day out specified. If the heartbeat task does not run on that specific day, the trigger method will never automatically execute.
